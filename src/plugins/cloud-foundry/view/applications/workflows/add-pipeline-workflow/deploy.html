Deploy App

In order to deploy your app, you must make a code change or trigger the delivery pipeline manually from your application's Delivery Logs view.

To automatically trigger the delivery pipeline, update the manifest.yml file in your source code repository. You can copy & paste the code below into the manifest.yml file in your repository. Helion Cloud Foundry uses the manifest file to set the application's name and route. Since the application and its route has been created for you, please ensure that the name, host and domain of your manifest file matches the example below. If the application name and route is not correct in the manifest file, two applications and routes will get created.

--- # We have created a temporary app in the gallery # We have created a delivery pipeline applications: # We have reserved a name/route for your app - name: {{ wizardCtrl.options.userInput.name }} host: {{ wizardCtrl.options.userInput.host }} domain: {{ wizardCtrl.options.userInput.domain.entity.name }} # We have provisioned and bound services services: - {{ service.name }}
